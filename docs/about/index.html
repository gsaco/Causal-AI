About

Built for people who want causal AI explained with sources, not vibes.

Causal AI Futures is a research atlas for the intervention era. It is designed for readers who want a clean map of the field, transparent evidence, and enough context to tell canonical work from fast-moving noise.

Editorial stance

The site optimizes for legibility, not maximal volume.

The goal is to make causal AI navigable without pretending the field is settled. We surface assumptions, disagreements, and open questions alongside the papers so readers can see where the field is strong and where it is still unstable.

Evidence first

Claims are anchored to arXiv IDs, with provenance and coverage exposed rather than hidden in the build.

Canonical plus rising

Landmarks stay visible while the navigator highlights recent momentum instead of flattening everything into one feed.

Static and inspectable

The site is built for static hosting, so the data products that power ranking and faceting remain easy to inspect.

How the evidence pipeline works

The system is opinionated by design: broad arXiv ingestion first, then relevance gates, rankings, and editorial framing.

01

Harvest

Query packs sweep the field through arXiv-first retrieval instead of a single topic query per page.

02

Filter

Off-topic papers are removed before persistence, and application feeds are built separately instead of mirroring the global corpus.

03

Rank

Canonical, rising, and recent scores give the navigator multiple lenses instead of a default recency sort.

04

Explain

Topic dossiers, trend pages, and application briefs translate the corpus into a field guide that can actually be read.

Mission

The goal is to make causal AI navigable without flattening the nuance. We surface signals, disagreements, and open problems alongside the papers, so readers can see where the field is converging and where it is not.

How evidence works

Claims are written in a <Claim> block with arXiv IDs. CI checks that evidence exists and validates citation coverage. Tendencies must include foundational anchors, new-wave signals, and debates.

How updates work

Weekday refreshes ingest new arXiv snapshots. A weekly digest summarizes movement, and provenance logs record harvest windows, query packs, and update history.
